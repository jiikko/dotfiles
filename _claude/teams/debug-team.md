# Debug Team

複数の仮説を並行して検証するデバッグチーム。forge の Ultra モードに相当する機能を Team Agents で実現する。

## チーム構成

### teammate: hypothesis-a
- **役割**: 仮説 A の検証（例: ロジックバグ、状態管理の問題）
- **使用エージェント**: debugger
- **モデル**: opus

### teammate: hypothesis-b
- **役割**: 仮説 B の検証（例: 非同期処理、タイミング問題）
- **使用エージェント**: crash-analyzer
- **モデル**: opus

### teammate: codebase-explorer
- **役割**: 関連コードの調査・影響範囲の特定
- **使用エージェント**: (Explore subagent)
- **モデル**: sonnet

## ワークフロー

1. **問題分析**: lead がバグの症状を整理し、複数の仮説を立てる
2. **並行検証**: 各チームメイトが独立して仮説を検証
3. **相互レビュー**: チームメイト同士で発見を共有・議論
4. **収束**: lead が最も有力な原因を特定し、修正を実施

## 使い方

```
デバッグチームを作成してください:
- hypothesis-a: [バグの症状] の原因が [仮説A] かどうか調査
- hypothesis-b: [バグの症状] の原因が [仮説B] かどうか調査
- codebase-explorer: [バグに関連するコード] の依存関係と影響範囲を調査
バグ: [バグの説明]
```

## 収束条件

- 複数の仮説が同じ根本原因を指す場合 → 高信頼度で修正
- 仮説が矛盾する場合 → 追加の検証ラウンドを実施
- 全仮説が棄却された場合 → 新しい仮説を立てて再度並行検証
