# レイヤーディレクトリに CLAUDE.md が無ければ問いかけるルール

## ルール

- **「レイヤーディレクトリ」で非自明な編集を行う前に、そのディレクトリに `CLAUDE.md` が存在するか確認する**
- **存在しない場合は、勝手に作成しない / 黙って続行しない。ユーザーに作成可否を一度だけ問いかける**
- 「作らない」「あとで」と言われたら、そのセッション中はそのディレクトリについて再度問わない
- 問いかけには「なぜ作成を提案するのか (責務 / ファイル数 / 推測される local convention)」を 2〜3 行添える。yes/no だけ聞かない

## 「レイヤーディレクトリ」の定義

以下のいずれかを満たすディレクトリ:

- 名前が責務を示す (`Models/`, `Views/`, `Services/`, `Tests/`, `internal/<feature>/` 等)
- 機能 / タブ / モジュールに対応する (`Auth/`, `Settings/` 等)
- Package / Module ルート (`Package.swift` / `go.mod` / `Gemfile` がある)
- 配下に source file 5 個以上 or 2 階層以上のサブディレクトリ

対象外: ビルド成果物 (`build/`, `node_modules/` 等)、自動生成 (`*.xcodeproj/` 等)、設定ファイル 1〜2 個だけの置き場、asset のみのディレクトリ。迷ったら「新しい人が把握に 5 分以上かかりそうか」で判定。

## 「非自明な編集」の定義 (= 問いかけのトリガ)

- ✓ 新規ファイル作成 / 30 行以上の変更 / 構造変更 (サブディレクトリ作成・ファイル移動) / 責務・規約・制約に関わる変更
- ✗ (問いかけ不要) typo・format 修正 / コメント追加 / import 追加 / 30 行未満の bug fix / ユーザーが scope を明示的に絞った軽微タスク

## 問いかけの形式

```
このディレクトリ (`<path>`) に CLAUDE.md がありません。以下の理由から作成を提案します。

- 配置されているファイル: <N> 個 (主なもの: <file1>, <file2>)
- 推測される責務: <一行>
- 触りそうな local convention の候補: <一行>

作成しますか?
1. 作る (内容案を提示してから書く)
2. 作らない (今は不要)
3. あとで判断する (このセッションでは聞かない)
```

「作る」と返答が来たら、まず **内容案 (section 構成と bullet 案) を 5〜10 行で提示し、合意を取ってから Write** する。いきなりフルテキストを書かない。

## 作成判定の最終ガード

問いかけ前に [`claude-md-maintenance.md`](claude-md-maintenance.md) の「作るタイミング」基準を満たすか自己確認する。**満たさないなら問いかけ自体をスキップする** (中身が単純で local convention が無いディレクトリで無駄な選択を強いない)。

## やること / やらないこと

- ✓ レイヤーディレクトリでの非自明な編集の前にチェックし、不在なら 3 択で問う
- ✓ 「作る」なら内容案 → 合意 → Write の順を守る
- ✗ 無断で作成する / 黙って続行する
- ✗ 軽微な変更で毎回聞く / 一度「作らない」と言われたものを同セッションで再度問う
